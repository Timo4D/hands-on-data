Tabular data – a universal data structure
Raw data: The data in the form in which it got originally aqcuired 
Such data has the form of a table (hence „tabular data“): 
Data points (=“object”) X Features
•
Data point: an individual observed example (“object”) within the dataset. (e.g., a given house)
•
Feature: an individual property that describes each data point (e.g., price)
Most commonly, the data contains values for the features of objects
For example:
•
Objects:    Houses
•
Features:  area, location, price,… of each house

A combination of several factors makes it difficult to directly identify patterns in the data
(1) Many data points
➢
Integration of large amount of information is required
•
E.g., data set with 27000 street blocks as data points
(2) Multiple features
➢
Patterns can be multi-parametric (multi-dimensional) →harder to discover, grasp and visualize
•
E.g., house price affected by multiple features (age, location, area,...) with interactions between them
(3) Variance
➢
Obscures patterns, unless the features that causing the variance are also taken into account
•
E.g., Price of houses with the same number of rooms can vary because of different house-ages. If 
house-age will not be considered, then the affect rooms on the price will be strongly obscured
(4) Noise
➢
Obscures patterns
•
E.g., price of houses with similar features can have fluctuations, due to measurement errors, decision 
noise, and other unknown affecting factors
